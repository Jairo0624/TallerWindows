LOS LENGUAJES DE BASES DE DATOS ORIENTADO A OBJETOS

Los lenguajes orientados a objetos de bases de datos son un conjunto de lenguajes y tecnologías que permiten modelar y gestionar datos de manera similar a como se hace en la programación orientada a objetos (POO). Estos lenguajes están diseñados para trabajar con sistemas de gestión de bases de datos orientados a objetos (ODBMS), que son sistemas de gestión de bases de datos que almacenan y recuperan datos en forma de objetos

Estos lenguajes y tecnologías permiten a los desarrolladores trabajar con datos de manera más intuitiva y eficiente, ya que pueden modelar datos en la base de datos de una manera similar a como lo harían en su código orientado a objetos

Algunos ejemplos de lenguajes orientados a objetos de bases de datos incluyen:

SQL3: SQL3 es una extensión de SQL (Structured Query Language) que agrega soporte para la manipulación de datos orientados a objetos. Permite el almacenamiento y consulta de datos complejos, como objetos, colecciones y herencia.

Java Database Connectivity (JDBC): JDBC es una API de Java que permite a las aplicaciones Java interactuar con bases de datos, incluyendo bases de datos orientadas a objetos. Proporciona una interfaz para conectar, consultar y modificar datos en bases de datos a través de objetos Java.

Object-Relational Mapping (ORM): Aunque no es un lenguaje en sí mismo, ORM es una técnica que permite a los desarrolladores trabajar con bases de datos orientadas a objetos de una manera más familiar, mapeando objetos en código a tablas en la base de datos.

Python Object-Relational Mapping (ORM) frameworks: En el mundo de Python, existen diversos ORM frameworks como SQLAlchemy, Django ORM, y Peewee, que permiten a los desarrolladores trabajar con bases de datos orientadas a objetos en Python.

Hay que darle un vistazo al lenguaje de SQL3

Ademas de lo ya explicado del SQL3 anteriormente estaba destinada a ampliar las capacidades del SQL estándar para trabajar con datos complejos y objetos. Puede funcionar para admitir tanto bases de datos relacionales como bases de datos orientadas a objetos

Este lenguaje destaca por permitir el uso de tipos de datos complejos, como objetos, colecciones y tipos definidos por el usuario.

Herencia: SQL3 incorpora el concepto de herencia, que permite a las tablas heredar propiedades y características de otras tablas.

Puedes realizar consultas SQL3 que recuperen objetos completos, en lugar de solo datos tabulares. Esto facilita la recuperación y manipulación de datos orientados a objetos en la base de datos.

Este lenguaje incluye soporte para transacciones y control de concurrencia, lo que garantiza la integridad y consistencia de los datos, incluso cuando se trabaja con objetos complejos.
